Sophos Anti-Virus for User-Managed Machines
Attention: You are only allowed to install SOPHOS on a computer that belongs to Concordia University.
Sophos antivirus software can be configured to automatically update itself. This page will explain the steps required to install and configure Sophos to keep itself up to date.

It is important to note that the auto-update feature is available ONLY on the SOPHOS software package created after June 2005.

The instructions provided below are for installing and configuring SOPHOS for Windows XP. For instructions on how to install SOPHOS on computers running an OS other than Windows XP, you are invited to read the SOPHOS user guides provided below.

  1. Visit http://www.encs.concordia.ca/aits/public/sophos/  to download SOPHOS for your platform, as well as, to obtain the required username and password to enable the auto-update feature. 
  1. Uninstall any previous version of SOPHOS that may be on your computer. 
    • Log onto your computer as Administrator, then navigate to the control panel. 
    • Double click on "Add and Remove Programs". 
    • Find the SOPHOS software and uninstall it.'
    • Reboot your computer.
  1. If you are installing SOPHOS on a computer running Windows, ensure that the "Client for Microsoft Networks" is installed. To check if it is or not:
    • Log onto your computer as Administrator, then navigate to the control panel.
    • Double Click on "Network Connections".
    • Right Click on "Local Area Connection" and select Properties.
    • In the pop-up window, you should find "Client for Microsoft Networks", if not, click "Install" and add the service.
    • You may be asked to reboot your computer.

  1. Start the installer program by double clicking the SOPHOS program you downloaded in step 1.
    • It is recommended to use the default install directories.

  1. During the installation procedure, you will be prompted for an "Update Source". It is important to fill this information accurately. The "Address" is OS dependant and is listed in the table immediately below the figure.  For example, if you are running Windows XP, the addr ess you would use is: "virus.encs.concordia.ca:8123/esxp/" as shown in the figure.  The "Username" and "Password" are obtained from https://www.encs.concordia.ca/aits/public/sophos/sec/. 


    • If you have a proxy server, select the "Access the update source via a proxy server" and follow the instructions

  1. After the installation has been completed, SOPHOS will be temporarily unavailable while it contacts the update server to download the latest anti-virus engine and IDEs. When the SOPHOS shield, found at the bottom right corner of your desktop becomes "blue", you are ready to proceed to the next step. If you see a "red X" on the shield, this means that your machine could not communicate with our server. This could happen if:
    • You are using a Firewall and have NOT given "iupdate.exe" access to the internet
    • You are not connected to your ISP
    • You typed the wrong address  (proceed to step 7)
    • You typed the wrong Username and/or Password (proceed to step 7)
  1. Configure SOPHOS. To do this, first launch the SOPHOS Application from the "Start Menu". Click on the "Configure" Menu.  It is recommended you configure the "On-access scanning" and the "Right click scanning" options. For more information, please refer to the SOPHOS user-guides.


  1. ENCS has two SOPHOS update servers. This is to ensure that one server is available at all times. Add the secondary update server, by selecting "Configure" -> "Updating.
    1. If you made a mistake in typing the primary update server address, username or password, correct this first.
    2. Select the "secondary server" tab, add the correct address and the same username, password combination you obtained earlier. Once again, the table below lists the correct address to use for the secondary server.
